## 二、失效场景发现

### 场景 1: 快速移动导致完全失效 ❌

**测试方法**:
- 手部在 1-2 秒内快速移动 10cm 以上距离
- 观察相对距离变化和日志输出

**观察现象**:
```
正常测距: Δ=2.3mm, 累积=45.6mm
正常测距: Δ=3.1mm, 累积=48.7mm
快速移动: Δ=5.8mm, 累积=54.5mm
失效瞬间: Δ=0.00mm, 累积=0.0mm  ← 突然归零
```

**失效原因分析**:

1. **相位解缠算法限制**
   - LLAP 基于相位差计算距离变化
   - 相位解缠算法假设: **相邻帧相位变化 < π**
   - 快速移动时违反此假设 → 解缠失败

2. **数学推导**:
   ```
   声波频率 f = 20kHz
   声速 v = 343 m/s
   波长 λ = v/f = 17.15 mm
   
   最大可检测速度:
   v_max = (λ/2) / Δt
         = 8.575 mm / 0.04s  (相位法帧率~25Hz)
         ≈ 214 mm/s
         ≈ 21.4 cm/s
   
   实际测试: 手部快速移动速度约 50-100 mm/s
   → 超出算法最大跟踪速度 → 失效
   ```

3. **代码验证**:
   ```kotlin
   // PhaseRangeFinder.kt 第 XXX 行
   // 相位解缠逻辑
   while (unwrappedPhase[i] - unwrappedPhase[i-1] > Math.PI) {
       unwrappedPhase[i] -= 2 * Math.PI
   }
   // ❌ 当相位变化 >> π 时，该算法会产生错误结果
   ```

### 场景 2: 非直线运动导致测量偏差 ⚠️

**测试方法**:
- 手部保持相对距离不变，但旋转一定角度（如 30°、45°）
- 观察相对距离数值变化

**观察现象**:
```
直线移动:   Δ=2.1mm, Δ=2.3mm, Δ=2.0mm  (稳定)
旋转30°:    Δ=1.2mm, Δ=0.8mm, Δ=1.5mm  (波动)
旋转45°:    Δ=-0.3mm, Δ=0.5mm, Δ=-0.1mm (异常负值)
```

**失效原因分析**:

1. **算法假设被违反**
   - LLAP 假设: 目标沿声波传播的**直线方向**运动
   - 旋转运动: 实际路径是**弧线**，不满足直线假设

2. **多径效应影响**
   ```
   直达波路径: 扬声器 → 手 → 麦克风  (主要信号)
   反射波路径: 扬声器 → 手 → 墙壁 → 麦克风  (干扰)
   
   旋转时: 手的角度改变 → 反射波相位关系改变
          → 干涉模式变化 → 测量误差
   ```

3. **图示说明**:
   ```
   直线运动 (✅ 准确):
   Speaker ----→ Hand ----→ Mic
           d1          d2
   Δd = d1 + d2 (符合算法假设)
   
   旋转运动 (⚠️ 偏差):
   Speaker ----→ Hand (旋转) ----→ Mic
           d1     ↺ θ°       d2
   实际路径 ≠ 直线距离 → 相位计算误差
   ```

## 四、理论依据

### 相位测距基本原理

1. **相位-距离关系**:
   ```
   相位差 Δφ = 2π × (2Δd / λ)
   
   其中:
   - Δd: 距离变化量
   - λ: 声波波长 (17.15mm @ 20kHz)
   ```

2. **解缠算法限制**:
   ```
   可检测最大相位变化: π rad
   对应最大距离变化: λ/4 ≈ 4.3 mm
   
   若 Δd > 4.3mm/帧:
   → 相位跳变超过π
   → 解缠算法失效
   → 输出归零或跳变
   ```

## 五、改进建议

### 针对快速移动失效:

1. **提高采样率**
   - 当前: 40ms/帧 (25 Hz)
   - 建议: 20ms/帧 (50 Hz)
   - 效果: 最大可跟踪速度提升 2 倍

2. **多频率冗余**
   - 使用 3-5 个不同频率同时测距
   - 通过投票或加权平均提高鲁棒性

3. **速度限制检测**
   ```kotlin
   if (abs(phaseChange) > MAX_PHASE_CHANGE) {
       // 超出跟踪能力，标记为"失效"而非输出错误值
       return INVALID_MEASUREMENT
   }
   ```

### 针对非直线运动偏差:

1. **IMU 传感器融合**
   - 结合加速度计/陀螺仪判断运动模式
   - 检测到旋转时降低 LLAP 权重

2. **能量法+相位法动态加权**
   ```kotlin
   // 检测到旋转或异常时，降低相位法权重
   val weight = if (isRotationalMotion) 0.3 else 0.7
   finalDistance = weight * phaseDist + (1-weight) * energyDist
   ```

## 六、总结

| 失效场景 | 触发条件 | 现象 | 根本原因 | 可否改进 |
|---------|---------|------|---------|---------|
| 快速移动 | 速度>21cm/s | 相对距离归零 | 相位解缠算法限制 | ✅ 可改进 (提高采样率) |
| 非直线运动 | 旋转>30° | 数值异常波动 | 违反直线运动假设 | ⚠️ 部分改进 (传感器融合) |
| 多径干扰 | 复杂环境 | 随机误差 | 反射波干扰 | ⚠️ 难改进 (算法固有限制) |

**结论**: 
- LLAP 在理想条件下精度高（~1-2mm）
- 但对运动模式敏感，需要配合能量法互补
- 当前融合策略已部分缓解问题，但仍需进一步优化
